# 滚动截图功能说明

## 问题描述

原有的截图功能只能捕获初始可见的页面内容，对于使用滚动加载（infinite scroll）的网页，无法捕获通过滚动动态加载的内容。这是因为原来的实现使用 CSS transform 来模拟滚动，不会触发真正的滚动事件。

## 解决方案

实现了真正的滚动加载功能，在截图之前先执行以下步骤：

1. **滚动加载**：逐步向下滚动页面，触发滚动加载事件
2. **等待加载**：每次滚动后等待一段时间，让新内容加载完成
3. **检测完成**：检查是否滚动到页面底部
4. **重新获取高度**：滚动后重新获取页面总高度
5. **执行截图**：使用更新后的页面高度进行完整截图

## 修改的文件

### 1. `/src/core/bg/tabs.js`
- 添加了 `scrollAndLoadDeferredContent()` 函数：处理滚动加载逻辑
- 添加了 `getPageHeight()` 函数：获取页面当前高度
- 添加了 `checkIfAtBottom()` 函数：检查是否滚动到底部
- 修改了 `captureTab()` 函数：在截图前先执行滚动加载

### 2. `/src/core/content/content.js`
- 添加了 `content.realScroll` 消息处理：执行真正的页面滚动
- 添加了 `content.getPageHeight` 消息处理：返回页面总高度
- 添加了 `content.isAtBottom` 消息处理：检查是否滚动到底部

## 工作流程

```
用户触发截图
    ↓
调用 scrollAndLoadDeferredContent()
    ↓
循环执行：
  1. 获取当前页面高度
  2. 滚动到接近底部
  3. 等待内容加载 (500ms)
  4. 检查是否到底部
    ↓ - 如果到底部
滚动回顶部
    ↓
使用更新后的页面高度进行截图
    ↓
返回完整截图
```

## 参数配置

- `scrollStep`: 每次滚动的距离（默认为窗口高度）
- `waitTime`: 每次滚动后等待时间（默认 500ms）
- `maxScrollAttempts`: 最大滚动尝试次数（默认 50 次）

## 特性

1. **自动检测滚动加载**：自动识别需要滚动加载的页面
2. **智能等待**：给页面充足的时间加载动态内容
3. **防止无限循环**：设置最大滚动次数限制
4. **兼容原有功能**：不影响现有的截图功能

## 测试建议

1. 测试常见滚动加载网站（如社交媒体、新闻网站）
2. 测试长页面的性能表现
3. 测试不同网络条件下的加载效果
4. 验证截图完整性

## 注意事项

- 滚动加载会增加截图时间，但能捕获完整内容
- 对于非常大的页面，可能需要调整等待时间参数
- 某些网站可能有反爬虫机制，可能需要额外处理
